## Spacing Scale

Declared values — all multiples of 4, sourced from `globals.css @theme`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline tag spacing, locale toggle separator |
| sm | 8px | Tag internal padding, compact nav items, card meta spacing |
| md | 16px | Default element spacing, card padding, section paragraph gap |
| lg | 24px | Section inner padding, timeline station gap |
| xl | 32px | Layout gaps, hero split column gap |
| 2xl | 48px | Major section breaks between sections |
| 3xl | 64px | Page-level vertical section spacing |
| 4xl | 96px | Hero top/bottom padding |

Exceptions:
- `4xl: 96px` — deliberate scale extension for page-level hero padding. Follows the doubling progression (32 → 48 → 64 → 96) as the next step for macro layout spacing. Already declared in `globals.css @theme` as `--spacing-4xl: 96px`. Standard set values (max 64px) are insufficient for full-viewport hero sections.
- Touch targets (nav links, locale toggle, hamburger, accordion rows): `min-h-[44px] min-w-[44px]` — WCAG/Apple HIG accessibility requirement, established pattern from `locale-toggle.tsx`
- Scroll-spy indicator: 2px left border, no spacing token needed

## Typography

All values sourced from `globals.css @theme`:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Label | 14px (`--font-size-label`) | 400 regular | 1.3 (`--line-height-normal`) | Tags, tech-stack chips, nav links, footer meta, FAQ question labels |
| Body | 16px (`--font-size-body`) | 400 regular | 1.6 (`--line-height-loose`) | Body paragraphs, timeline station copy, FAQ answers, card descriptions |
| Heading | 20px (`--font-size-heading`) | 600 semibold | 1.15 (`--line-height-snug`) | Section headings, project card titles, timeline station job titles |
| Display | clamp(28px, 4vw, 36px) (`--font-size-display`) | 600 semibold | 1.15 (`--line-height-snug`) | Hero name "Tobias Ludwig", section hero labels |

Weights in use: 400 (regular) and 600 (semibold) — exactly 2 weights.

Additional rule: `--line-height-relaxed: 1.4` is available for intro paragraphs (timeline lead-in, hero value proposition) where loose 1.6 feels too open.

## Color

All values sourced from `globals.css @theme`:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F9FAFB` (`--color-surface`) | Page background, section backgrounds |
| Secondary (30%) | `#FFFFFF` (`--color-surface-raised`) + `#F3F4F6` (`--color-surface-subtle`) | Cards (project cards, metric cards), sticky header background, FAQ accordion background |
| Accent (10%) | `#2563EB` (`--color-accent`) | See reserved-for list below |
| Accent hover | `#1D4ED8` (`--color-accent-hover`) | Accent elements on hover/focus |
| Text primary | `#111827` (`--color-text-primary`) | All body text, headings |
| Text secondary | `#6B7280` (`--color-text-secondary`) | Meta labels, timeline dates, tag text, secondary nav items |
| Border | `#E5E7EB` (`--color-border`) | Card borders, section dividers, accordion separators, timeline vertical line |
| Destructive | `#DC2626` (`--color-destructive`) | Not used in Phase 2 (no destructive actions) |

**Accent reserved for:**
1. Active scroll-spy nav link (text + left-border indicator)
2. "CV herunterladen" CTA button (background fill)
3. "Details →" project card link text
4. Focus-visible outline (all interactive elements — already global via `:focus-visible` rule)
5. Active locale toggle state is text-primary (NOT accent) — per existing `locale-toggle.tsx` pattern

Accent is NOT used for: section headings, body text, tag backgrounds, timeline dots (use border color), inactive nav links.

## Layout Contracts

### Sticky Header
- Height: 64px (--spacing-3xl)
- Background: `--color-surface-raised` (#FFFFFF) with 1px bottom border `--color-border`
- Desktop: left-aligned logo/name + right-aligned nav links + locale toggle
- Mobile (< 768px): left logo, right hamburger button (44x44px touch target)
- Nav links: 14px label weight, text-secondary default, text-primary + accent left-border 2px on active section
- z-index: above all content

### Hero Section
- Padding: 4xl top/bottom (96px), md horizontal on mobile, xl on desktop
- Desktop: 2-column split — left ~55%, right ~45%
- Left column: Display-size name, heading-size role label (text-secondary), body value proposition, CTA button
- Right column: 2x2 grid of metric cards, each card surface-raised with border, lg padding
- Mobile: stacked — text above, metrics below

### Career Timeline
- Vertical line: 1px solid `--color-border`, centered on timeline dots
- Station dots: 8px circle, border `--color-border` fill, `--color-surface-raised` center
- Station layout: dot + content side-by-side. Date label: 14px text-secondary. Title: 16px semibold. Body: 16px body line-height-relaxed.
- Station gap: lg (24px)
- Left gutter for dot: md (16px) from edge

### Project Cards (Desktop: 3-column grid, Mobile: 1-column)
- Card: surface-raised background, 1px border, rounded-lg (8px radius), lg padding
- Card gap: md (16px)
- Tags row: flex-wrap, gap-xs, sm padding each tag, 12px/label text-secondary
- "Details →" link: label size, accent color, no underline, hover accent-hover

### Skills Section
- Category heading: heading size, semibold, text-primary
- Tags: flex-wrap row, gap-sm between tags, gap-lg between categories
- Tag: surface-subtle background, border, rounded-full, sm vertical padding, md horizontal padding, 14px label

### FAQ Accordion
- Each item: full-width row, heading border-bottom `--color-border`
- Question row: body size, semibold, 44px min-height, flex justify-between + chevron icon
- Chevron: text-secondary, rotates 180deg when open (CSS transform only — no Framer Motion in Phase 2)
- Answer: body text, relaxed line-height, md padding bottom, text-secondary

### Footer
- Background: surface-subtle
- Content: "Built with Next.js, Tailwind CSS, Framer Motion, TypeScript, next-intl" — label size, text-secondary
- Separator: border-top `--color-border`
- Padding: 2xl vertical

### Project Detail Page
- Back link: "← Zurück / ← Back" at top, label size, accent color
- Three sections (Problem / Ansatz / Ergebnis): heading-size section title, body text
- Tech tags: same Tag component as ProjectCard
- GitHub link (optional): label size, accent color, external icon

## Accessibility Contract

- All interactive elements: `min-h-[44px] min-w-[44px]` touch targets
- All images (hero, metric cards): descriptive `alt` attributes
- Landmark regions: `<header>`, `<main>`, `<nav>`, `<footer>`, `<section aria-labelledby>`
- FAQ accordion: `aria-expanded` on trigger, `aria-controls` pointing to answer panel
- Hamburger: `aria-expanded`, `aria-label="Menü öffnen / schließen"`
- Scroll-spy: does not move focus — visual only
- Color contrast: text-primary (#111827) on surface (#F9FAFB) — passes WCAG AA
- Color contrast: text-secondary (#6B7280) on surface (#F9FAFB) — passes WCAG AA (4.6:1)
- Color contrast: accent (#2563EB) on surface (#F9FAFB) — passes WCAG AA for large text
